CRITICAL PATH RULE — READ THIS BEFORE TOUCHING ANY FETCH() CALL
  HTML files live at: DATABASE/GRANDE HTML/LANDING PAGE/
  PHP files live at:  DATABASE/
  Therefore from HTML: ../../endpoint.php reaches DATABASE/endpoint.php

  CORRECT:   fetch('../../login.php', ...)
  WRONG:     fetch('../../DATABASE/login.php', ...)   ← 404, double-path error

  This is the single most common bug in this codebase. Every fetch() you write
  or modify must use ../../  with NO DATABASE/ segment after it.

BUG FIX 4 — documents table missing UNIQUE constraint
────────────────────────────────────────────────────────────────────────────────
Run this SQL in phpMyAdmin → SQL tab ONE TIME:

  ALTER TABLE documents
    ADD UNIQUE KEY unique_user_doc (user_id, doc_type);

This is required for the ON DUPLICATE KEY UPDATE in upload.php to work
correctly. Without it, re-uploading a document creates duplicate rows
instead of replacing the existing one.

If you get "Duplicate entry" error when running it, first run:
  DELETE d1 FROM documents d1
  INNER JOIN documents d2
    ON d1.user_id = d2.user_id AND d1.doc_type = d2.doc_type
   AND d1.id > d2.id;
Then re-run the ALTER TABLE.

        up.uni_name, up.country, up.website,

NEW FILE 4 — DATABASE/intake_stats.php
────────────────────────────────────────────────────────────────────────────────
Purpose: Public endpoint (NO auth required) that returns anonymous aggregate
stats about the student pipeline. Used on the public Intake Dashboard page
to show universities what the pipeline looks like before they pay.

GET handler — NO require_auth():
  Run these 6 queries and return them all in one response:

  1. total_students:
     SELECT COUNT(*) FROM users WHERE role='student' AND is_active=1

  2. verified_count:
     SELECT COUNT(*) FROM student_profiles WHERE verified='verified'

  3. docs_complete_count:
     SELECT COUNT(DISTINCT user_id) FROM documents
     WHERE user_id IN (SELECT id FROM users WHERE role='student')
     GROUP BY user_id HAVING COUNT(*) >= 4
     (wrap in subquery to get total count)

  4. by_nationality: (top 10)
     SELECT nationality, COUNT(*) as count FROM student_profiles
     WHERE nationality IS NOT NULL AND nationality != ''
     GROUP BY nationality ORDER BY count DESC LIMIT 10

  5. by_major: (top 10)
     SELECT major, COUNT(*) as count FROM student_profiles
     WHERE major IS NOT NULL AND major != ''
     GROUP BY major ORDER BY count DESC LIMIT 10

  6. by_budget:
     SELECT budget, COUNT(*) as count FROM student_profiles
     WHERE budget IS NOT NULL AND budget != ''
     GROUP BY budget ORDER BY count DESC

  Return:
  {
    success: true,
    total_students: N,
    verified_count: N,
    docs_complete_count: N,
    by_nationality: [{nationality, count}, ...],
    by_major: [{major, count}, ...],
    by_budget: [{budget, count}, ...]
  }

  CORS note: This endpoint is public so add at the top before require_once:
    header('Access-Control-Allow-Origin: *');

SECTION 9 — ABSOLUTE CODING RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

These rules must never be broken. Any code that violates them will cause
bugs that are hard to diagnose and will waste hours of debugging.

RULE 1 — API PATHS
  All fetch() calls must use:   '../../endpoint.php'
  Never use:                    '../../DATABASE/endpoint.php'
  This is a double-path error that causes silent 404s.

RULE 2 — NO BUNDLER
  Do not add any import or export statements.
  Do not reference node_modules, package.json, or any npm packages.
  All libraries come from CDN scripts already in the HTML <head>.

RULE 3 — PHP RESPONSES
  Every PHP file must respond with respond([]) or fail([]).
  Never use echo, print, or die with a string directly.
  Always set Content-Type: application/json (done by db.php).

RULE 4 — CREDENTIALS
  Every fetch() to a PHP endpoint must include: credentials: 'include'
  Without this, the BA_SESSION cookie is not sent and auth fails.

RULE 5 — PREPARED STATEMENTS
  Every SQL query that uses user input MUST use prepared statements
  with bind_param(). Never concatenate user input into SQL strings.

RULE 6 — TARGETED EDITS
  Do not rewrite entire files. Locate the exact function or component
  and apply the minimum change needed. This prevents introducing new bugs
  into already-working code.

RULE 7 — COMPONENT ORDER
  In market.html, all React components must be defined before they are
  used. If component A uses component B, B must appear earlier in the file.
  NotificationsDropdown must appear before TopBar.
  IntakeDashboard must appear before App.

RULE 8 — ERROR HANDLING
  Every fetch() must have a .catch() or try/catch that fails gracefully.
  Never let an unhandled promise rejection crash the entire React app.
  For user-facing errors, show a message in the UI — never alert().

RULE 9 — ADMIN AUTH
  All new admin endpoints must call require_admin() not require_auth().
  require_admin() internally calls require_auth() and also checks role=admin.
  An authenticated student or university must NOT be able to reach admin
  endpoints even if they construct the request manually.

RULE 10 — TCPDF PATH
  XAMPP includes TCPDF. Require it in certificate.php like this:
  $tcpdf_path = 'C:/xampp/htdocs/tcpdf/tcpdf.php';
  if (!file_exists($tcpdf_path)) {
      // Try composer path
      $tcpdf_path = __DIR__ . '/../../vendor/tecnickcom/tcpdf/tcpdf.php';
  }
  if (!file_exists($tcpdf_path)) fail('TCPDF not found. Install via composer require tecnickcom/tcpdf', 500);
  require_once $tcpdf_path;

  If TCPDF is not installed, generate a simple HTML-to-text receipt instead
  and save it as a .txt file — do not fail silently.
